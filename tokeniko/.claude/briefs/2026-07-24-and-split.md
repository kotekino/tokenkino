## The Captain's design (the spine)

A compound utterance whose root is a PLAIN AND over independent claims is «TWO sentences treated
separately, but part of the same interaction»: «the cat is a mammal and pigs fly because Z» →
1. «the cat is a mammal because Z» — agree/learn; 2. «pigs fly because Z» — WTF → a speakup that
NAMES the offending conjunct («why are you saying that pigs fly because Z? that contradicts what
I know…»). Today the evaluator grounds each clause but FOLDS the truths into one verdict (fuzzy
AND = min → the absurd conjunct drags the true one; the reaction can't name which part offends).

**Fork b (ruled): split at EVALUATION/REACTION, never at ingestion.** The memory item stays
WHOLE — true history be it — and `brain/thinking.py` fans per-conjunct verdicts into
per-conjunct ideas. The evaluator already knows the per-clause truths; the change is to expose
them to the reaction layer instead of folding them away.

## The split criterion (STRICT — the carve-outs are load-bearing)

Split ONLY when ALL of:
- the root operator is a **plain AND** — never `AND[contrast]` (the M1 contrast flag pairs the
  conjuncts semantically) and never the cause-annotated ANDs (`AND[cause:*]` — one claim + its
  reason, not two claims);
- the conjunct clauses have **DISTINCT subject atoms** (the consistency kernel's atom clustering
  is the discriminator) — same-subject coordination NEVER splits: «the cat is dead and alive» is
  ONE X∧¬X contradiction and `evaluator_classifyForm` needs the whole form to see it.

A shared subordinate («because Z», a THAT-attitude, a marker clause) DISTRIBUTES to every
conjunct — the `compiler_evaluateCoordinates._inherit_shared` cousin (the author's own early
machinery: subject+copula already distribute; subordinates join them). Investigate whether the
compile already attaches the subordinate at the coordination root or on the last conjunct, and
distribute at the SPLIT site (the reaction layer), not by changing the compiled zip — the stored
zip stays exactly what the compiler produced.

## Build (the shape; refine against the code and report deviations)

1. **The evaluation seam** (`lib/core/evaluation_harness.py` / `brain/thinking.py`): where the
   assertion path today takes ONE `EvaluatorResult` for the item, detect the split criterion on
   the zip's operator tree; when it fires, evaluate each qualifying conjunct subtree as its own
   statement (each with the distributed subordinates) and carry a list of per-conjunct results
   (verdict + the conjunct's own rendered text + derivation) alongside the whole-item result.
2. **The reaction fan** (`brain/thinking.py`): per-conjunct eval:* classification — each conjunct
   spawns its own ideas (eval:true / eval:false / eval:unknown / the teach path / speakup), each
   idea's payload carrying the CONJUNCT's text so the reply names it («…that pigs fly…»).
   The interaction stays one: same source memory item id on every idea (the «same
   subject/interaction» requirement — nothing new to store).
3. **The teach path**: a true-and-novel conjunct is teachable on its own (the existing
   `_taught_candidate` machinery per conjunct) — the false sibling never poisons it.
4. **Questions are OUT OF SCOPE** — the split applies to the assertion path only; a compound
   question keeps today's behavior.
5. **Tests** (sibling style, sandbox): the flagship specimen (mixed true+absurd → two reactions,
   the speakup naming the absurd conjunct; the true conjunct teachable) · same-subject

   `AND[cause:*]` do not split · subordinate distribution (both conjuncts carry «because Z» in
   their evaluated form) · single-clause items unchanged. Full gate green (all passed, 1 xfailed
