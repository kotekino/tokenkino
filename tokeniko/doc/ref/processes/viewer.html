A · API compilation pipeline as-is

How a natural-language sentence becomes a stored, queryable memory item. A client POSTs a sentence to an axiom/definition/theorem resource; the *Service parses it (spaCy+Stanza → recursive AST, with word-sense & identity disambiguation against the KB), compiles it to the flat TKLLC + the numeric TKZip (running the graph-constrained genus untangle in-pass), gates it against the contradiction guard (logic-is-sacred), and stores it. The one fork: a form that folds to 0 under every crisp assignment is rejected (422); tautologies and contingent statements pass.

B · /evaluate as-is

Judging a claim (grounding), read-only — stores nothing. The statement compiles the same way as A, then the parser-free evaluator runs: first the intra-statement contradiction kernel (logic-is-sacred); then each clause is decided by whichever path proves it — relational (is_a subsumption / tiered-disjointness), forward-chaining (rules + facts), or a near-exact geometric definition match (affirm only); the clause truths fold through the operator tree; finally a geometric best-match against active axioms/theorems. The active KB is loaded once (fingerprint-cached).

C · Brain coordinator tick as-is

The single loop, dynamic priority routing: each tick drains Actions before Priorities before Thinking — the reactive path wins, thinking is the background filler. One bounded unit per phase, then a cooperative yield; only true idle takes the long sleep. Event-driven interruption falls out for free (a new idea/action is picked up next tick).

D · Reactive loop — perceive → think → decide → act as-is senses-out dry-run

The whole reactive arc across three phases. Thinking picks the liveliest speaker's fresh item and branches on mood: a question is answered (not believed); a declarative is evaluated, fanned into eval:* ideas via the behavior rules, cross-checked against the speaker's priors, and (if forward-chain-true) silently materialized. Priorities scores each idea on urge × feasibility and collapses siblings to one reflex per decision. Actions executes: INTERNAL by the brain, outward via senses (dry-run today).

E · Wondering — autonomous derivation as-is

The idle filler that grows knowledge silently. First KB-wondering: seed the chainer from everything tokeniko knows, keep only genuinely-new (≥2-premise) conclusions, render + POST them as first-class theorems (min-trust, provenance) — this is how «I exist» was born in-loop. When that's converged, fall to memory-wondering: an associative driver (KB-delta senses pick touched memories) and a throttled random drift, processing one past item per tick and materializing anything newly-derivable. Convergence is by construction (a materialized conclusion lands in held).

F · KB ingestion & the tiers (definitions-as-rules) as-is

How the vocabulary feeds reasoning. At compile time every definition's genus is untangled (graph-constrained WSD → sense faithful to the subject's taxonomy). Batch extractors then mine the clean definitions into two low-trust, revocable tiers — is_a edges and property rules — each strictly gated against the bedrock graph. The reasoning readers split cleanly: grounding uses the rich definitions; chaining uses bedrock ∪ the derived tiers, held honest by the step-4 net. Per decision A, the noisy WordNet auto-rules are not bulk-applied — quality comes from curated fuel.

G · Senses I/O + public window as-will-be

tokeniko's I/O to the world — mostly projectual (drawn ahead to force forward-consistency; purple = not built yet). Inbound (planned): a Discord/ATProto listener routes each message to memory, with the open question of overheard-vs-directed. Outbound (built, dry-run): the brain mints a channel=discord action; senses resolves the recipient, decompiles the raw stance to fluent English, and delivers (dry-run by default until a live sender lands). Public window (mock): the brain's actions loop one-way-publishes transmissions + hardware/cycle stats to a separate public Atlas.
